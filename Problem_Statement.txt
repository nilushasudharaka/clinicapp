  Project: MediCare – Clinic Appointment Management System

1. INTRODUCTION
---------------------------------------------------------------
The healthcare industry continues to face significant challenges
in managing patient-doctor interactions efficiently. Traditional
clinic management systems rely heavily on manual processes such
as paper-based appointment booking, handwritten prescriptions,
and in-person billing — all of which are time-consuming, error-
prone, and inaccessible to modern patients who expect digital,
on-demand services.

MediCare is proposed as a full-stack mobile clinic management
system designed to digitise and streamline core clinical
workflows for patients, doctors, and administrators.


2. PROBLEM DESCRIPTION
---------------------------------------------------------------
Clinics and small-to-medium healthcare providers currently face
the following operational challenges:

  (a) Appointment Booking Inefficiency
      Patients cannot book appointments remotely. They must
      physically visit or call the clinic, leading to long wait
      times, missed appointments, and administrative overload.

  (b) Lack of Centralised Patient Records
      Doctor notes, prescriptions, billing records, and
      appointment history are stored separately with no unified
      view for the patient or the doctor.

  (c) Poor Communication Between Stakeholders
      When a doctor reschedules or an admin cancels an
      appointment, the patient receives no timely notification,
      leading to confusion and missed visits.

  (d) Manual Billing Processes
      Invoice generation, tracking payment status, and
      communicating outstanding bills to patients is done
      manually, causing delays and errors.

  (e) Prescription Accessibility
      Patients have no digital record of their prescriptions
      and must physically return to the clinic to retrieve them.

  (f) No Emergency Contact Integration
      Clinic systems do not provide quick access to emergency
      contacts or ambulance services within the same platform.


3. PROPOSED SOLUTION
---------------------------------------------------------------
MediCare addresses the above problems through a cross-platform
mobile application (React Native / Expo) backed by a RESTful
API (Node.js / Express) and a MongoDB database.

  Key Solution Modules:

  • Patient Module
    - Self-registration and secure login
    - Online appointment booking with doctor selection
    - Real-time notifications for appointment updates,
      billing, and prescription events
    - Access to bills, prescriptions, and reports from
      each appointment card

  • Doctor Module
    - Dashboard showing today's appointments and pending tasks
    - Ability to add prescriptions and request reschedules
    - View patient history per appointment

  • Admin Module
    - Full appointment lifecycle management
      (confirm / cancel / reschedule)
    - Doctor and patient record management
    - Invoice generation linked to completed appointments
    - Billing status management

  • Public / Home Module
    - Accessible landing page without login required
    - Emergency contact quick-dial (ambulance, hospital lines)
    - Hospital information, services directory, and contact form


4. OBJECTIVES
---------------------------------------------------------------
  1. Develop a secure, role-based mobile application for
     patients, doctors, and administrators.

  2. Enable patients to book, view, and manage appointments
     from their mobile device at any time.

  3. Implement a real-time notification system that alerts
     patients when appointments are confirmed, cancelled,
     rescheduled, billed, or prescribed.

  4. Create a digital billing system that generates invoices
     for completed appointments and allows online payment
     confirmation.

  5. Provide doctors with a streamlined dashboard to manage
     their daily appointments and add prescriptions digitally.

  6. Build an admin portal to oversee all clinic operations
     from a single interface.

  7. Expose a well-documented RESTful API that can be extended
     or integrated with third-party healthcare services.


5. SCOPE
---------------------------------------------------------------
  IN SCOPE:
  ✔ Patient registration, login, and profile management
  ✔ Appointment booking, confirmation, cancellation,
    and rescheduling
  ✔ Doctor dashboard with appointment management
  ✔ Admin dashboard with full CRUD operations
  ✔ Prescription creation and patient-side viewing
  ✔ Invoice generation and payment status tracking
  ✔ Patient notification system (appointment, billing,
    prescription events)
  ✔ Public home page with emergency contacts and
    contact-us enquiry form

  OUT OF SCOPE:
  ✘ Real-time payment gateway integration (placeholder only)
  ✘ Video or telemedicine consultation
  ✘ Insurance claim processing
  ✘ Lab report file upload (placeholder for future)

---------------------------------------------------------------

  Database  : MongoDB (via Mongoose ODM)
  Auth      : JWT (JSON Web Tokens) with refresh token support
  Storage   : Local file system (doctor profile photos)
  API Style : RESTful JSON API
